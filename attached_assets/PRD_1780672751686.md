# PRD — Sinal · WhatsApp Intelligence + CRM

**Produto:** Dashboard que transforma o WhatsApp do Bruno (DMs + grupos, já salvos no Supabase) em inteligência acionável: o que te trazem no privado, que pautas rolam nos grupos, quem fala de você/seus produtos, e um CRM de contatos com tasks e histórico.

**Nome provisório:** Sinal (sinal > ruído). Definir antes do build.

**Construir em:** Replit (backend + frontend). Banco: Supabase existente. IA: **OpenAI API** (key direta).

**Princípio nº 1:** o sistema **só lê** o WhatsApp. Ele interpreta e organiza; quem age no zap é o Bruno. Zero automação de envio.

**Data:** 2026-06-05 · **Status:** PRD final, escopo travado para MVP.

---

## 1. Visão e estratégia

Hoje 85k mensagens caem num Supabase e morrem lá. Este produto lê tudo, classifica via IA e devolve num dashboard que responde:

- **Privado:** o que me trazem, quanto tempo invisto, quem me procura, quais convites, o que não respondi.
- **Grupos:** o que estão discutindo, quais pautas, o que está trending em vários grupos ao mesmo tempo.
- **Menções:** falaram de mim ou dos meus produtos? Elogio, crítica, objeção de venda, recomendação.
- **CRM:** quem é cada pessoa, com tasks e histórico, sincronizável com o Google Contacts.

**Construir multi-tenant desde o dia 1.** O Bruno é o tenant nº 1 ("coçar a própria coceira"), mas a infra nasce isolada por conta pra virar SaaS depois. A fronteira: Dados+CRM+Menções viram multi-tenant com `tenant_id`+RLS (barato). A **ingestion do WhatsApp de terceiros** (cada cliente conectar o próprio número) é feature de produto cara e fica **fora deste MVP** — mas o schema já nasce pronto pra recebê-la.

## 2. Escopo do MVP

### 2.1. Dentro — 6 áreas
1. **Overview** — KPIs do dia + fila de não-respondidas + trending.
2. **Privado** — categorias, temas/pautas, tempo investido (estimado), convites, volume por pessoa, não-respondidas.
3. **Grupos** — pautas cross-grupo, o que estão discutindo, trending, resumo por grupo.
4. **Menções** — quem fala de você/produtos, por sentimento e tipo.
5. **Contatos (CRM)** — só DM (521) + promoção manual, com tasks, histórico, labels, sync Google.
6. **Salvos & Tasks** — central que junta os itens salvos (🔖) + gestão de todas as tasks.

Mais: pipeline de enriquecimento (Whisper + OCR + classificação), mecânica única **salvar**, e **drawers de contexto** (todo insight desce até as mensagens reais que o originaram).

### 2.2. Fora do MVP (fase 2+)
- **Perguntas sem boa resposta** nos grupos (cara e imprecisa — exige a IA julgar se uma resposta foi "boa").
- **Ações múltiplas** por insight (virar task/FAQ/conteúdo/associar separados) — substituídas por uma só: salvar.
- **Contatos de grupo no CRM** — só DM entra; grupo não vira contato automático.
- **Ingestion de WhatsApp de terceiros** (onboarding de conexão por tenant).
- Responder/postar pelo dashboard, sugestão de resposta por IA, app mobile.

## 3. Arquitetura

```
REPLIT
 ├─ Scheduled Deployment  → job de enriquecimento (incremental 15min + backfill 1x)
 ├─ Autoscale Deployment  → API REST (métricas, CRM, tasks, saved, menções, sync Google)
 ├─ Static/Autoscale      → frontend React (dashboard)
 └─ Secrets: SUPABASE_URL/KEY · OPENAI_API_KEY · GOOGLE_OAUTH · WHATSAPP_OWNER

Supabase (fonte + enrichment, RLS por tenant)
OpenAI API (classificação=modelo mini · Whisper=áudio · vision=OCR · resumo=modelo maior)
Google People API (Contacts: import auto + export aprovado)
```

**Sem Hermes.** O backend chama a OpenAI key direto (pay-per-use). Agente ≠ backend de SaaS; classificação em massa é ETL determinístico.

## 4. Modelo de dados (tabelas novas)

Todas com `tenant_id uuid` + RLS por tenant. DDL de referência:

```sql
-- Tenancy
create table tenants (id uuid primary key default gen_random_uuid(), name text, created_at timestamptz default now());
-- usuários via Supabase Auth (auth.users) + profile ligando user → tenant

-- CRM
create table contacts (
  id uuid primary key default gen_random_uuid(), tenant_id uuid not null,
  display_name text, email text, description text, primary_phone text,
  google_resource_name text, dominant_category text, last_interaction_at timestamptz,
  source text default 'dm',           -- 'dm' (automático) | 'promoted' (puxado de grupo na mão)
  created_at timestamptz default now(), updated_at timestamptz default now()
);
create table contact_identifiers (
  id uuid primary key default gen_random_uuid(), tenant_id uuid not null,
  contact_id uuid references contacts(id) on delete cascade,
  phone text not null, source text, unique(tenant_id, phone)
);
create table labels (id uuid primary key default gen_random_uuid(), tenant_id uuid, name text, color text);
create table contact_labels (contact_id uuid references contacts(id) on delete cascade, label_id uuid references labels(id) on delete cascade, primary key(contact_id,label_id));

-- Enriquecimento por mensagem (1:1)
create table message_enrichment (
  message_id text primary key references whatsapp_messages(message_id), tenant_id uuid not null,
  chat_type text, category text, sentiment text, topics text[],
  is_question boolean, requires_reply boolean, summary text,
  model_used text, processed_at timestamptz default now()
);

-- Mídia processada (Whisper / OCR)
create table media_assets (
  message_id text primary key references whatsapp_messages(message_id), tenant_id uuid not null,
  kind text, extracted_text text, status text default 'pending', model_used text, processed_at timestamptz
);

-- Pautas / temas agregados (privado e grupo)
create table topics (
  id uuid primary key default gen_random_uuid(), tenant_id uuid not null,
  label text, scope text,              -- 'private' | 'group'
  period_start date, period_end date,
  person_count int, message_count int, trend text, summary text,
  created_at timestamptz default now()
);
create table topic_messages (topic_id uuid references topics(id) on delete cascade, message_id text references whatsapp_messages(message_id));  -- prova / drill-down
create table topic_groups (topic_id uuid references topics(id) on delete cascade, chat_id text, message_count int);  -- cross-grupo: "em N grupos"

-- Menções a entidades monitoradas
create table monitored_entities (
  id uuid primary key default gen_random_uuid(), tenant_id uuid not null,
  name text, type text,                -- 'person' | 'product' | 'competitor'
  aliases text[]                       -- ["Bruno","Okamoto","cara do TEDx"]
);
create table mentions (
  id uuid primary key default gen_random_uuid(), tenant_id uuid not null,
  message_id text references whatsapp_messages(message_id), entity_id uuid references monitored_entities(id),
  mention_type text,                   -- elogio | critica | objecao | recomendacao | indireta | neutra
  sentiment text, created_at timestamptz default now()
);

-- Grupos (metadados derivados)
create table groups (chat_id text, tenant_id uuid, name text, message_count int, last_activity_at timestamptz, primary key(tenant_id, chat_id));
create table group_digests (id uuid primary key default gen_random_uuid(), tenant_id uuid, chat_id text, period_start date, period_end date, summary text, top_excerpts jsonb, created_at timestamptz default now());

-- Tasks (central; ligada a contato OU a uma menção/insight)
create table tasks (
  id uuid primary key default gen_random_uuid(), tenant_id uuid not null,
  contact_id uuid references contacts(id) on delete set null,
  title text, direction text,          -- 'mine' | 'theirs' | 'internal'
  source_message_id text references whatsapp_messages(message_id),  -- de onde veio (nullable)
  due_at timestamptz, done boolean default false, done_at timestamptz,
  created_at timestamptz default now()
);

-- Salvos (mecânica única do botão 🔖)
create table saved_items (
  id uuid primary key default gen_random_uuid(), tenant_id uuid not null,
  kind text,                           -- 'pauta' | 'prova_social'
  source_type text, source_id text,    -- topic | mention + id
  text text, created_at timestamptz default now()
);
```

**Views materializadas** (refresh pelo Scheduled Deployment) para o dashboard: métricas por categoria, ranking de contatos, fila de não-respondidas, topics cross-grupo.

## 5. Pipeline de enriquecimento (OpenAI)

Roda no Scheduled Deployment, idempotente (só processa mensagem sem `message_enrichment`). Ordem por mensagem:

1. **Mídia** (se ≠ texto): áudio → **Whisper** ; imagem → **vision (OCR)** → `media_assets.extracted_text` (entra como input da classificação).
2. **Classificação** → **modelo pequeno OpenAI** (ex. `gpt-5-mini`/`nano` ou `gpt-4.1-mini`) retornando JSON: `{category, sentiment, topics[], is_question, requires_reply, summary}`. Batch de 15–25 msgs, prompt de sistema cacheado.
3. **Menções** → checa contra `monitored_entities` (match por alias **+ detecção semântica**: "o cara do TEDx" = Bruno sem citar o nome). Grava `mentions` com tipo + sentimento.
4. **Grava** `message_enrichment`.

**Jobs agregados** (não por mensagem):
- **Pautas** (`topics`): clustering dos `topics[]` + nomeação por LLM, contagem por pessoa/grupo. Para grupos, calcula co-ocorrência cross-grupo (`topic_groups`). Mantém `topic_messages` pra reconstruir a prova nos drawers.
- **Resumo de grupo** (`group_digests`): modelo maior, semanal.
- **Tempo/dia respondendo**: heurística de **sessão de conversa** (janelas contíguas com troca). É **estimativa**, rotulada como tal na UI — não há dado exato de tempo de digitação.

**Backfill:** execução única em lotes (84.858 msgs). **Filtro de owner** em todo o pipeline.

## 6. As 6 áreas (telas)

Referência visual: ver `mockup.html` (dark-first, denso, estilo Linear/Attio).

**6.1. Overview** — 4 KPIs (recebidas, não-respondidas, tempo/dia *estimado*, resp. média) + fila de não-respondidas + trending + volume + sentimento.

**6.2. Privado** — KPIs de gestão (recebidas/enviadas, tempo/dia estimado, convites, pessoas ativas) · **temas/pautas específicos** (não buckets vagos) com botão 🔖 · trending pautas · distribuição por categoria · **🎙️ Convites & oportunidades** (bloco dedicado: podcast/palestra/entrevista/parceria com status) · volume por pessoa · não-respondidas.

**6.3. Grupos** — **📡 pautas cross-grupo** (o mesmo tema em N grupos, com 🔖) · o que estão discutindo · trending · tabela de grupos por pautas. Detalhe do grupo: resumo IA → pautas → trechos que pegaram fogo → "também rolando em outros grupos". **Sem temperatura/picos/membros** (descartado por irrelevante).

**6.4. Menções** — KPIs (menções, elogios, objeções, recomendações) · filtro por entidade (você / cada produto) · **⭐ recomendações** (venda orgânica) · lista por tipo (elogio/crítica/objeção·lead/indireta) com 🔖. Configuração de entidades monitoradas em Conexões.

**6.5. Contatos (CRM)** — painel (tasks abertas, followups atrasados, pra arquivar, em dia) · tabela (tasks, labels, categoria, último followup com atraso). Perfil: dados editáveis (nome/email/whatsapp/labels/descrição) · **tasks com check** · histórico completo · sync Google (botão). Só DM + promovidos.

**6.6. Salvos & Tasks** — central. **Tasks** (todas num lugar, com contato, direção, atraso, check, filtros) + **Salvos** (pautas pra conteúdo + prova social, filtros). Destino do botão 🔖.

**Drawers de contexto** — clicar em pauta ou task abre painel lateral: pauta → por que virou pauta, onde rola, **trechos reais (a prova)**; task → contato, **mensagem de origem**, notas, ações. Princípio: nenhum número é beco sem saída.

## 7. CRM e regras de relacionamento

- **CRM = só os 521 com DM** + contatos **promovidos manualmente** (`source='promoted'`). Os ~2.500 que só aparecem em grupo **não** entram (evita lixão).
- **Promoção:** ao ver alguém relevante numa menção (ex: lead que perguntou "vale o preço?"), o Bruno puxa pro CRM num clique → cria `contact` com `source='promoted'`.
- **Tasks:** `contact_id` opcional — nascem de um contato OU de um insight (menção/crítica sem contato). Check conclui (`done`).
- **Sync Google:** import automático (casa por telefone, preenche `google_resource_name`); export/atualização **só com aprovação por contato**, nunca em lote.

## 8. API REST (Autoscale) — endpoints principais

```
# Inteligência
GET /api/metrics/private/{categories|volume|trending|unanswered|sentiment|top-contacts|time-spent|invites}
GET /api/topics?scope=group&crossgroup=true        # pautas cross-grupo
GET /api/topics/:id                                # drill-down: grupos + trechos reais
GET /api/groups · GET /api/groups/:chatId/digest
GET /api/mentions?entity=&type=                    # menções por entidade/tipo

# CRM + Tasks + Salvos
GET/PATCH /api/contacts · POST /api/contacts/:id/promote · /merge · /labels
GET /api/contacts/:id/messages
GET/POST/PATCH /api/tasks?filter=late|mine|theirs  # toggle done
GET/POST/DELETE /api/saved                          # botão 🔖
POST /api/contacts/:id/google/export                # export aprovado

# Config
GET/POST /api/entities                              # entidades monitoradas
```

## 11. Segurança e privacidade

- **RLS por tenant** em todas as tabelas novas; service key só no servidor.
- Secrets no Replit. Filtro `WHATSAPP_OWNER` em todas as queries.
- Google OAuth escopo mínimo (`contacts`); export só com aprovação.
- Single-user no MVP (auth simples), mas multi-tenant-ready.
- Sistema só-leitura no WhatsApp → zero risco de envio errado / ToS de automação.
